Working with APIs & HTTP

Make HTTP requests, handle JSON, authenticate with API keys, and build robust retry logic for production AI apps.

What Is an API and Why Does AI Need Them?

An API (Application Programming Interface) is a contract between two programs. You send a request with data; the server returns a response. Every AI capability you use in production — GPT-4, Claude, Stable Diffusion, Google Vision — is accessed via an HTTP API. You cannot build AI apps without knowing how HTTP works.

The key mental model: Every API call is just: (1) format a request (headers + body), (2) send it to a URL, (3) parse the JSON response. That's it.

The requests Library

import requests

# GET request -- retrieve data
response = requests.get("https://api.github.com/users/python")
print(response.status_code)  # 200 = success
data = response.json()       # parse JSON response
print(data["public_repos"])

# POST request -- send data
payload = {"username": "alice", "action": "login"}
resp = requests.post("https://httpbin.org/post", json=payload)
print(resp.json()["json"])  # echo of what we sent

# Always check status
response.raise_for_status()  # raises exception if 4xx/5xx

Headers and Authentication

# API keys go in headers -- never in URLs
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
response = requests.get("https://api.example.com/data", headers=headers)

# Query parameters
params = {"q": "python AI", "limit": 10, "page": 1}
response = requests.get("https://api.example.com/search", params=params)
# Results in: /search?q=python+AI&limit=10&page=1

Working with JSON

import json

# Parse JSON string to Python dict
data = json.loads('{"name": "Alice", "score": 95}')
print(data["name"])  # Alice

# Convert Python dict to JSON string
json_str = json.dumps({"name": "Alice", "score": 95}, indent=2)

# Save to file
with open("results.json", "w") as f:
    json.dump(data, f, indent=2)

with open("results.json") as f:

Error Handling for APIs

import requests, time

def call_api(url, headers, retries=3):
    for attempt in range(retries):
        try:
            resp = requests.get(url, headers=headers, timeout=10)
            resp.raise_for_status()
            return resp.json()
        except requests.exceptions.Timeout:
            print(f"Timeout on attempt {attempt+1}")
            time.sleep(2 ** attempt)  # exponential backoff
        except requests.exceptions.HTTPError as e:
            if resp.status_code == 429:  # rate limited
                time.sleep(10)
            else:
                raise
    raise RuntimeError("All retries failed")
Never hardcode API keys in your code. Use environment variables: import os; key = os.environ["OPENAI_API_KEY"]. Never commit keys to GitHub.

2

Use query parameters

# Search GitHub for Python AI repos
params = {"q": "python AI tutorial", "sort": "stars", "per_page": 5}
resp = requests.get("https://api.github.com/search/repositories", params=params)
results = resp.json()["items"]
for repo in results:
    print(f"{repo['full_name']:40s} stars={repo['stargazers_count']}")

Save API results to JSON

import json

resp = requests.get("https://api.github.com/users/python")
data = resp.json()

# Save selected fields
summary = {"name": data["name"], "repos": data["public_repos"], "followers": data["followers"]}
with open("github_python.json", "w") as f:
    json.dump(summary, f, indent=2)
print("Saved:", summary)
